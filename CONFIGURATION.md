# Hardcore Systems configuration guide

Файл настроек мода создается после первого запуска по пути:

```text
Documents\Klei\OxygenNotIncluded\mods\Local\ONI.HardcoreSystems\config\hardcore_systems.json
```

После изменения настроек полностью перезапусти Oxygen Not Included. Файл должен оставаться валидным JSON: строки в кавычках, логические значения только `true` или `false`, дробные числа через точку.

Если настройка выходит за допустимый диапазон, мод считает конфиг некорректным и загружается с пресетом `Off`.

## Быстрый безопасный старт

Для ручной настройки ставь:

```json
"Preset": 5
```

`5` означает `Custom`. Это не обязательно для загрузки, но так сразу видно, что значения изменены вручную.

Самые полезные настройки для баланса:

- `Mining.Enabled`
- `Mining.YieldMultiplier`
- `Duplicants.Enabled`
- `Duplicants.ExperienceMultiplier`
- `Duplicants.CaloriesMultiplier`
- `Diseases.Enabled`
- `Diseases.SeverityMultiplier`
- `IndustrialHeat.Enabled`
- `IndustrialHeat.HeatMultiplier`
- `Power.GeneratorRebalanceEnabled`
- `Power.SolarPanelGenerationHeatEnabled`
- `Power.OverloadHeatEnabled`

## Preset

`Preset` и `Global.LastSelectedPreset` хранят выбранный пресет.

Значения:

| Число | Название | Смысл |
|---:|---|---|
| `0` | `Off` | Все gameplay-модули выключены. |
| `1` | `VanillaPlus` | Легкое усложнение. |
| `2` | `Hard` | Среднее усложнение. |
| `3` | `Extreme` | Сильное усложнение. |
| `4` | `Insane` | Максимальный встроенный пресет. |
| `5` | `Custom` | Ручные значения. |

Пресеты задают стартовые значения, но если ты редактируешь файл руками, меняй конкретные секции ниже.

## Global

| Поле | Что делает | Безопасно |
|---|---|---|
| `Language` | Маркер языка для будущей локализации/интерфейса. | `en` или `ru`. Сейчас почти не влияет. |
| `LogLevel` | Маркер уровня логов. | Лучше оставить `Info`; текущий runtime-логгер стартует с `Info`. |
| `LastSelectedPreset` | Последний выбранный пресет. | Обычно держать равным `Preset`. |

## World

Эти настройки относятся к новым мирам и worldgen-пресетам. Они не уменьшают уже существующие сохранения.

| Поле | Что делает | Безопасно |
|---|---|---|
| `Enabled` | Включает использование настроек worldgen-размера. | `true` или `false`. |
| `AsteroidSize` | Семейство размера астероида. | `Vanilla`, `Half`, `Quarter`, `Eighth`. |
| `IsReadOnlyAfterWorldCreation` | Напоминание, что размер мира нельзя менять для уже созданного сохранения. | Лучше оставить `true`. |

Размеры пресетов:

| Размер | Base game | Spaced Out classic / Vanilla-style |
|---|---:|---:|
| `Half` | `180 x 272` | `170 x 268` |
| `Quarter` | `128 x 192` | `120 x 190` |
| `Eighth` | `90 x 136` | `84 x 134` |

`Half`, `Quarter`, `Eighth` означают примерную долю площади, а не деление каждой оси напрямую.

## Mining

| Поле | Что делает | Безопасный диапазон |
|---|---|---|
| `Enabled` | Включает уменьшенный выход ресурсов при копании. | `true` или `false`. |
| `YieldMultiplier` | Множитель массы, выпадающей из выкопанных клеток. | `0.01` - `1.00`. |

Примеры:

| Значение | Эффект |
|---:|---|
| `1.00` | Ванильный выход ресурсов. |
| `0.50` | Выпадает 50% от обычной массы. |
| `0.25` | Выпадает 25% от обычной массы. |
| `0.01` | Почти ничего не выпадает; очень жесткий режим. |

## Duplicants

| Поле | Что делает | Безопасный диапазон |
|---|---|---|
| `Enabled` | Включает изменения опыта и калорий дубликантов. | `true` или `false`. |
| `ExperienceMultiplier` | Множитель получаемого опыта. | `0.05` - `10.00`. |
| `CaloriesMultiplier` | Множитель расхода калорий. | `0.10` - `10.00`. |

Для сложности обычно:

- `ExperienceMultiplier` ниже `1.00`: дубликанты медленнее получают навыки.
- `CaloriesMultiplier` выше `1.00`: нужно больше еды.

Осторожные значения:

| Настройка | Мягко | Сложно | Очень сложно |
|---|---:|---:|---:|
| `ExperienceMultiplier` | `0.80` | `0.60` | `0.35` |
| `CaloriesMultiplier` | `1.10` | `1.25` | `1.50` |

## Diseases

| Поле | Что делает | Безопасный диапазон |
|---|---|---|
| `Enabled` | Включает дополнительные штрафы от болезней. | `true` или `false`. |
| `SeverityMultiplier` | Сила дополнительных штрафов. | `0.00` - `0.95`. |

Примеры:

| Значение | Эффект |
|---:|---|
| `0.00` | Дополнительные штрафы отключены. |
| `0.10` | Легкое усиление болезней. |
| `0.30` | Заметные штрафы. |
| `0.50` | Сильно опаснее ванили. |
| `0.80` | Очень жестко. |

## Power

| Поле | Что делает | Безопасно |
|---|---|---|
| `GeneratorRebalanceEnabled` | Меняет профиль мощности/тепла у топливных генераторов: ручной, угольный, дровяной, торфяной, водородный, газовый, нефтяной. | `true` или `false`. |
| `SolarPanelGenerationHeatEnabled` | Солнечные панели выделяют тепло пропорционально текущей генерации. | `true` или `false`. |
| `OverloadHeatEnabled` | Провода и мосты получают тепловую энергию при перегрузочном уроне. | `true` или `false`. |

Это включатели без числового диапазона. Если надо мягче, выключай отдельный пункт.

## IndustrialHeat

| Поле | Что делает | Безопасный диапазон |
|---|---|---|
| `Enabled` | Включает масштабирование тепла у затронутых не-генераторных построек. | `true` или `false`. |
| `HeatMultiplier` | Множитель тепловыделения. | `0.00` - `10.00`. |

Примеры:

| Значение | Эффект |
|---:|---|
| `0.00` | Дополнительное масштабирование фактически выключено. |
| `1.00` | Базовое значение мода. |
| `1.50` | На 50% больше тепла. |
| `2.00` | В 2 раза больше тепла. |
| `10.00` | Максимум валидатора; для обычной игры слишком агрессивно. |

## Diagnostics

| Поле | Что делает | Безопасный диапазон |
|---|---|---|
| `Enabled` | Включает легкие диагностические метрики в логах. | `true` или `false`. |
| `ReportIntervalSeconds` | Как часто писать диагностический отчет. | `1` - `600` секунд. |

Для обычной игры держи `false`. Для отладки удобно `true` и `60`.

## RadiativeHeat

Секция зарезервирована под будущий модуль. В текущем runtime она не зарегистрирована как активная gameplay-система.

| Поле | Что делает | Безопасный диапазон |
|---|---|---|
| `Enabled` | Зарезервированный включатель. | `true` или `false`, но сейчас лучше `false`. |
| `SimulationIntervalSeconds` | Зарезервированный интервал симуляции. | `0.20` - `600`. |
| `RadiationFactor` | Зарезервированный коэффициент. | Сейчас лучше оставить `0`. |

## FluidPressure

Секция зарезервирована под будущий модуль. В текущем runtime она не зарегистрирована как активная gameplay-система.

| Поле | Что делает | Безопасный диапазон |
|---|---|---|
| `Enabled` | Зарезервированный включатель. | `true` или `false`, но сейчас лучше `false`. |
| `LiquidFillThreshold` | Зарезервированный порог заполненности жидкостью. | `0.01` - `0.50`. |
| `SimulationIntervalSeconds` | Зарезервированный интервал симуляции. | `0.20` - `600`. |

## Готовые ориентиры

Мягкий режим:

```json
"Mining": { "Enabled": true, "YieldMultiplier": 0.50 },
"Duplicants": { "Enabled": true, "ExperienceMultiplier": 0.80, "CaloriesMultiplier": 1.10 },
"Diseases": { "Enabled": true, "SeverityMultiplier": 0.10 },
"IndustrialHeat": { "Enabled": true, "HeatMultiplier": 1.25 }
```

Сложный режим:

```json
"Mining": { "Enabled": true, "YieldMultiplier": 0.25 },
"Duplicants": { "Enabled": true, "ExperienceMultiplier": 0.60, "CaloriesMultiplier": 1.25 },
"Diseases": { "Enabled": true, "SeverityMultiplier": 0.30 },
"IndustrialHeat": { "Enabled": true, "HeatMultiplier": 1.50 }
```

Очень жесткий режим:

```json
"Mining": { "Enabled": true, "YieldMultiplier": 0.10 },
"Duplicants": { "Enabled": true, "ExperienceMultiplier": 0.35, "CaloriesMultiplier": 1.50 },
"Diseases": { "Enabled": true, "SeverityMultiplier": 0.50 },
"IndustrialHeat": { "Enabled": true, "HeatMultiplier": 1.75 }
```

## Что лучше не трогать

- `SchemaVersion`, если нет миграции конфига.
- `RadiativeHeat`, пока модуль не реализован.
- `FluidPressure`, пока модуль не реализован.
- `World.AsteroidSize` для уже созданного сохранения: это не изменит размер существующего мира.
