# v0.4 Electrical Overload Audit

## Scope

This audit covers the corrected v0.4 overload thermal damage model implemented in 0.4.3.

Implemented:

- keep ONI's native overload detection and damaged-object selection;
- patch `BuildingHP.DoDamage(int)` after vanilla damage;
- require overload damage metadata from `BuildingHP.GetDamageSourceInfo()`;
- support normal `Wire` and bridge-like `WireUtilityNetworkLink` conductors;
- calculate target temperature as `0.90 * Element.highTemp` in Kelvin;
- calculate energy from `mass * SHC * max(0, targetK - currentK)`;
- inject energy through `SimMessages.ModifyBuildingEnergy` using the structure-temperature sim handle;
- deduplicate repeated callbacks for the same object in the same Unity frame.

Not implemented:

- wire resistance;
- voltage drop;
- power routing;
- load-flow solving;
- constant heating of healthy wires;
- changing native wire wattage limits.

## Runtime Findings

`ElectricalUtilityNetwork.UpdateOverloadTime(float dt, float watts_used, List<WireUtilityNetworkLink>[] bridgeGroups)` is the native overload path. It waits until overload time exceeds 6 seconds, chooses either a `WireUtilityNetworkLink` from bridge groups or a `Wire` from wire groups, then triggers `BuildingHP.DamageSourceInfo` on that exact `GameObject`.

Observed overload damage metadata:

```text
source = STRINGS.BUILDINGS.DAMAGESOURCES.CIRCUIT_OVERLOADED
popString = STRINGS.UI.GAMEOBJECTEFFECTS.DAMAGE_POPS.CIRCUIT_OVERLOADED
statusItemID = Db.Get().BuildingStatusItems.Overloaded.Id
effect = spark_damage_kanim
damage = 1
```

`BuildingHP.OnDoBuildingDamage(object)` stores the metadata, calls `DoDamage(int)`, then plays pop/effects. A postfix on `DoDamage(int)` can read `GetDamageSourceInfo()` and still address the same damaged object.

## Material Data

The damaged object's `PrimaryElement` provides:

```text
PrimaryElement.Element.highTemp
PrimaryElement.Element.specificHeatCapacity
PrimaryElement.Mass
PrimaryElement.Temperature
```

`Element.highTemp` is treated as the material phase transition temperature for this stage.

## Thermal Injection

The adapter uses:

```text
GameComps.StructureTemperatures.GetHandle(gameObject)
GameComps.StructureTemperatures.GetPayload(handle).simHandleCopy
SimMessages.ModifyBuildingEnergy(simHandle, deltaKDTU, minTemperatureK, maxTemperatureK)
```

The pure calculator returns DTU. `ModifyBuildingEnergy` accepts kDTU, so runtime divides by 1000.

## Risks

- If a conductor building has no registered structure-temperature sim handle at the exact damage callback, 0.4.3 skips heat and logs a rate-limited warning instead of falling back to direct temperature assignment.
- Deduplication uses `GameObject.GetInstanceID()` plus `UnityEngine.Time.frameCount`; a stricter simulation-tick event ID was not found in the public overload callback.
- Joint plates are covered when represented as `WireUtilityNetworkLink`; an in-game overload test is still needed to confirm whether ONI selects them as damage targets.

## Verification

Unit coverage added for:

- Kelvin target formula;
- energy calculation from mass and SHC;
- already-hot conductors;
- invalid mass, SHC, melting temperature, NaN, and infinity.
