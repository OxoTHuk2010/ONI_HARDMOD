# v0.7 Worldgen Asteroid Size Audit

## Scope

Version 0.7.10 implements experimental asteroid-size variants as data-only worldgen assets.

Implemented:

- base-game `Vanilla` clusters from `StreamingAssets/worldgen/clusters`;
- Spaced Out `SpacedOutVanillaStyle` classic clusters from `StreamingAssets/dlc/expansion1/worldgen/clusters`;
- generated `Half`, `Quarter`, and `Eighth` variants as separate `HardcoreSystems_*` cluster/world YAML files;
- no vanilla YAML is modified in place;
- no runtime shrinking is applied to existing saves.

Not implemented in 0.7.0:

- Spaced Out `SpacedOutStyle` mini-cluster shrinking;
- DLC2/DLC4/DLC5 custom asteroid families;
- world temperature presets;
- new biomes, story traits, POI rules, rocket rules, geyser output changes.

## Size Model

`Half`, `Quarter`, and `Eighth` are area targets, not per-axis cuts:

- `Half`: axis scale `sqrt(0.50)`, area about 50%;
- `Quarter`: axis scale `0.50`, area about 25%.
- `Eighth`: axis scale `sqrt(0.125)`, area about 12.5%.

Half keeps the maps large enough for start placement, world borders, subworld rings, and regular POI/template placement in current testing.

Quarter keeps the compact size target, but removes inherited `worldTemplateRules`. In-game testing showed that quarter-size maps repeatedly failed on guaranteed template placement even after removing inherited subworld count guarantees and world trait pressure. The 0.7.2 strategy prioritized successful compact-map generation over preserving mandatory geyser, warp, Gravitas, and story/POI templates.

In 0.7.3 Quarter also replaces inherited full-size `unknownCellsAllowedSubworlds` with compact placement rules. The generated layout keeps the printing pod starter biome to the start subworld only, uses a single compact ring for starter-adjacent resources, mixes non-surface/non-magma biome pockets after that, limits magma/lava content to the `AtDepths` band, and limits space/regolith content to `AtSurface`. This is intended to make Quarter playable rather than just loadable.

In 0.7.4 Quarter tightened biome footprint further, but `OverworldMinNodes: 24` could abort generation when ONI produced fewer layout points on quarter-size maps. Version 0.7.5 lowered the hard minimum to `OverworldMinNodes: 1`, matching Klei tiny-world presets, but in-game testing showed `OverworldDensityMin: 80` and `OverworldDensityMax: 90` still produced too few very large regions on 20-25% maps.

Version 0.7.6 reverses that tuning direction for Quarter and uses `OverworldDensityMin: 6`, `OverworldDensityMax: 10`, `OverworldAvoidRadius: 2`, and `OverworldMaxNodes: 800`. This is intended to produce many smaller PowerTree pockets, closer to the target of biome areas no larger than about 30x30 cells. Magma/core rules remain limited to the `AtDepths` band. Surface handling is split: actual space subworlds are pinned with `AtTag AtSurface`, while surface/regolith crust subworlds are limited to `DistanceFromTag AtSurface` at distance `1..1`, reducing sideways regolith placement and neutronium-closed tops.

Version 0.7.7 also stops treating Spaced Out vanilla-style Quarter worlds as vanilla-only biome lists. For generated `dlc/expansion1` Quarter worlds, the generator adds a shared compact DLC biome pool after the source world's `subworldFiles`. The pool is checked against real `expansion1` YAML files before insertion and currently includes selected swamp, rust, radioactive, wasteland, frozen, ocean, forest, marsh, jungle, oil, and barren subworlds. Base-game `worldgen` Quarter worlds are left without `expansion1::` subworld references so they remain valid when DLC content is unavailable.

Version 0.7.8 adds an even smaller `Eighth` preset using the same compact worldgen pipeline as Quarter. `Eighth` uses lower PowerTree density defaults (`OverworldDensityMin: 4`, `OverworldDensityMax: 7`) and minimum axes of `80x112` so it can target roughly half of Quarter's area without collapsing below a plausible starter footprint. Compact presets also reduce `WorldBorderThickness` from `5` to `3`; this intentionally targets thinner outer border bands first, while leaving individual subworld `borderOverride` rules intact unless in-game testing shows they are the source of excessive abyssalite walls.

Version 0.7.9 extends DLC biome mixing to Half Spaced Out vanilla-style worlds. Half still keeps inherited template rules and does not compact every subworld, but generated `dlc/expansion1` Half worlds add the shared DLC biome pool, compact only the starting subworld, and replace the full-size fill rules so the starter biome is not used as a large fallback region. The Half DLC fill list puts DLC candidates before vanilla candidates while keeping vanilla candidates available; this is intended to keep vanilla area roughly balanced instead of letting it dominate the map. Base-game `worldgen` Half worlds still avoid `expansion1::` references.

Version 0.7.10 removes inherited hard `worldTemplateRules` from generated Half Spaced Out worlds and replaces them with optional `TryOne` resource rules. This is a stability correction after in-game testing showed Half generation failures with the expanded DLC biome pool and compact starter. Base-game Half worlds keep their conservative inherited template rules.

Version 0.7.10 also thins compact subworld borders. Compact generated subworlds for Quarter, Eighth, and forced Half starts now set `borderSizeOverride` to `min: 0` and `max: 2`, and lower `borderOverridePriority` to `5`. This keeps biome separation and some abyssalite/rocky borders, but makes thick five-cell walls much less likely to dominate compact maps.

Generated worlds scale `worldTraitScale` by the area ratio.

Generated worlds remove inherited `minCount` and `maxCount` guarantees inside `subworldFiles` and `subworldMixingRules`. The 0.7.0 attempt scaled those counts down to at least `1`, but ONI still failed reduced-world generation when a guaranteed subworld could not be placed. The reduced maps now rely on `unknownCellsAllowedSubworlds` and the normal layout rules to choose what fits.

Generated reduced worlds also remove inherited `worldTraitRules` and `subworldMixingRules`, and set `disableWorldTraits: true`. This is intentional for 0.7.1: asteroid size is the mechanic being tested, while vanilla world traits and mixing rules add extra placement pressure that is tuned for full-size maps.

Quarter generated worlds also remove inherited `worldTemplateRules`. Half keeps those rules in base-game generated worlds, while Half Spaced Out generated worlds use the same optional resource-rule strategy as compact presets. Quarter adds back relaxed optional `TryOne` rules from curated water and mixed geyser/vent lists, with up to six placement attempts total. The rules are intentionally optional, so reduced maps can gain resource variety when space exists without making template placement a hard generation requirement.

Half DLC worlds do not use Quarter's compact PowerTree defaults. The only compact Half subworld is the starting subworld in generated `dlc/expansion1` worlds; the rest of the biome pool stays based on source Klei subworld definitions to preserve Half as the conservative size preset.

Quarter generated worlds use compact generated subworld variants instead of vanilla subworld definitions. These compact subworlds set `avoidRadius: 2`, lower `minChildCount`, relax `dontRelaxChildren`, force `pdWeight: 0.2` on every compact subworld, force `borderSizeOverride: 0..2`, lower `borderOverridePriority` to `5`, and remove regular subworld `features` while keeping the `centralFeature` used by start locations. Quarter also removes unresolved `(Mixing*)` placeholders after `subworldMixingRules` are removed and drops inherited `overridePower` entries from Quarter `subworldFiles`.

## Safety Decisions

Base-game vanilla clusters contain one asteroid placement, so their generated variants replace that single world reference.

Spaced Out classic clusters contain start worlds plus rocket worlds, outer moonlets, Temporal Tear/worldMixing rules, and story-critical placements. In 0.7.0 only `locationType: StartWorld` references are replaced with generated smaller worlds. Other worlds remain vanilla to avoid breaking story-critical generation.

## Generated Assets

Generation command:

```powershell
.\tools\generate-worldgen-assets.ps1 -OniGameDir 'D:\SteamLibrary\steamapps\common\OxygenNotIncluded'
```

Generated asset roots:

```text
src/HardcoreSystems.Mod/worldgen
src/HardcoreSystems.Mod/dlc/expansion1
```

The installer copies both roots to the local mod folder.

## Validation Limits

This pass validates the generated files structurally by source transformation and build packaging only. Full worldgen correctness still requires in-game seed smoke tests because ONI generation failures depend on seed, DLC state, and template placement.

Recommended smoke-test matrix:

- base game `Hardcore SandstoneDefault Half`;
- base game `Hardcore SandstoneDefault Quarter`;
- Spaced Out classic `Hardcore VanillaSandstoneCluster Half`;
- Spaced Out classic `Hardcore VanillaSandstoneCluster Quarter`;
- at least 5 different seeds per selected cluster before expanding support to mini-clusters.
